**Kong은 "API와 AI 트래픽이 지나다니는 고속도로 톨게이트(관문)"를 만드는 회사**입니다. 원래는 **API 게이트웨이**로 유명해진 실리콘밸리 회사인데, 요즘은 **"AI 에이전트 시대(Agentic Era)의 연결(connectivity) 인프라 회사"**로 자기 정체성을 바꾸는 중입니다. 행사 제목 `Agentic Era World Tour`가 바로 그 새 정체성을 전 세계에 알리는 투어입니다.

> 핵심 한 줄: **"예전엔 앱끼리 API로 연결했다면, 이제는 AI 에이전트끼리·AI 모델끼리 연결해야 하는데, 그 연결을 안전하게 통제·관리·과금하는 게 Kong이 하려는 일"**

## A. API와 API 게이트웨이가 뭔지부터 (비전공자용)

**API(Application Programming Interface)** = 프로그램끼리 정보를 주고받는 약속/창구. 예: 배달앱이 카드사에 "이 결제 승인해줘"라고 요청하는 통로가 API.

문제는, 회사가 커지면 이런 API가 **수백~수천 개**로 늘어납니다 (벤츠는 1,500개 이상 관리). 그러면:
- 누가 이 API를 쓸 수 있지? (인증/보안)
- 한 명이 1초에 1만 번 때리면? (트래픽 제어)
- 누가 얼마나 썼지? (분석/과금)
- 갑자기 서버가 죽으면? (부하 분산)

이걸 **API마다 따로 만들면 지옥**입니다. 그래서 모든 API 앞에 **"공항 보안검색대 + 안내데스크" 같은 단일 관문**을 두는데, 그게 바로 **API 게이트웨이**입니다. 들어오는 모든 요청을 여기서 검사·인증·기록·분배합니다.

**Kong Gateway**가 바로 이 관문 제품이고, 업계에서 가장 빠르고 유명한 것 중 하나입니다.
- NGINX 엔진 기반, **노드당 초당 5만 건 이상** 처리
- 코드의 89%가 **Lua 언어**(OpenResty/Nginx 기반)
- **오픈소스(Apache 2.0 라이선스)** → 누구나 무료로 쓸 수 있고, 그래서 폭발적으로 퍼짐 (2020년 기준 다운로드 1.5억 회 돌파)
- REST, gRPC, GraphQL, WebSocket, Kafka 등 다양한 프로토콜 지원
- **플러그인 60개+** (Plugin Hub)로 기능 확장 (인증, 트래픽 제어, AI 기능 등)

## D. 경쟁사 대비 차별점

주요 경쟁자: **Google Apigee, Salesforce MuleSoft, AWS API Gateway, Tyk**

Kong이 내세우는 차별점:
- **성능**: 자체 벤치마크에서 Apigee보다 **약 31배 높은 처리량**(5.4만 TPS vs 1,750 TPS)이라고 주장 ⚠️ *자체 벤치마크라 참고만*
- **배포 단순함**: Kong은 단일 노드 + Postgres/Cassandra로 시작 가능 vs Apigee는 최소 9노드 필요
- **멀티클라우드/쿠버네티스 네이티브**: Apigee가 구글 클라우드에 묶여 있는 것과 대조
- **오픈소스 + 거대 커뮤니티**: Tyk보다 큰 커뮤니티
- **AI 특화**: 경쟁사보다 빠르게 AI/에이전트 기능을 붙임

한계도 솔직히: 데이터 변환 레이어가 약하고(레거시 SOAP↔REST 변환엔 부적합), 통합 과금 기능이 상대적으로 약하다는 평가 → 그래서 2025년 OpenMeter 인수로 보강.

## E. ⭐ 'Agentic Era' / AI Gateway 전략 (행사의 핵심!)

행사가 전부 이 얘기입니다. 꼭 이해하고 가세요.

### 왜 AI가 게이트웨이를 새로 필요로 하나?
지금까지는 **앱 ↔ 앱**이 API로 통신했습니다. 그런데 AI 시대엔:
1. **앱 → AI 모델**(LLM 호출): ChatGPT, Claude, Gemini 같은 모델을 부름
2. **AI 에이전트 → 도구/데이터**: 에이전트가 사내 API·DB를 호출 → 여기서 **MCP**가 표준으로 등장
3. **AI 에이전트 → AI 에이전트**(A2A): 에이전트들끼리 서로 대화하며 협업

이게 다 **통제 불능의 새로운 트래픽**입니다. 누가 어떤 모델을 얼마나 쓰는지, 비용은 얼마인지, 민감정보(PII)가 새는지, 악의적 프롬프트 주입은 없는지... Kong은 **"이걸 통제하는 새로운 인프라 계층이 필요하다"**고 주장하며 그 자리를 노립니다.

### 핵심 키워드 3개
- **MCP (Model Context Protocol)**: AI 에이전트가 외부 도구·데이터에 연결되는 **새로운 표준 규약**. Kong은 이걸 "에이전트 시대의 핵심 표준"으로 보고 강하게 베팅
- **A2A (Agent-to-Agent)**: 에이전트 간 통신
- **"API call 경제 → Token 경제"**: 예전엔 API 호출 횟수가 단위였다면, 이제는 AI **토큰**이 비용·가치의 단위. Kong은 **"AI 토큰의 흐름을 안전하게 연결·통제·수익화한다"**고 포지셔닝 ("The AI Connectivity Company")

### Kong AI Gateway가 실제로 하는 일
- **멀티 LLM 라우팅**: OpenAI, Anthropic, Google Gemini, AWS Bedrock, Azure, Mistral 등 **수천 개 모델을 하나의 통일된 API로** — 코드 수정 없이 모델 교체 가능
- **비용/지연시간 기반 라우팅, 시맨틱 라우팅, 시맨틱 캐싱** (같은 질문은 캐시로 답해 비용 절감)
- **토큰 예산/한도 관리**, 컨텍스트 윈도우 거버넌스
- **보안**: PII(개인정보) 자동 차단 (20개+ 카테고리, 12개 언어), HashiCorp Vault 등 비밀관리 연동
- **MCP 자동 생성**: 기존 Kong이 관리하던 일반 REST API를 **코드 한 줄 안 짜고 MCP 서버로 변환** + OAuth 2.1 중앙 인증

### 성능 주장
자체 벤치마크 기준 경쟁 AI 게이트웨이(LiteLLM, Portkey 등)보다 **LiteLLM 대비 859% 빠름, Portkey 대비 228% 빠름**이라고 주장 (시맨틱 캐싱·토큰 인지 속도제한 덕분). ⚠️ *자체 수치라 참고용*

## 용어 치트시트
- **API 게이트웨이**: 모든 API 요청이 지나는 단일 관문(보안검색대)
- **LLM**: ChatGPT·Claude 같은 거대언어모델
- **MCP**: AI 에이전트가 도구·데이터에 붙는 표준 규약 (행사 핵심 키워드)
- **A2A**: 에이전트끼리의 통신
- **에이전틱(Agentic)**: AI가 스스로 도구를 쓰며 일을 수행하는 방식
- **토큰**: AI가 처리하는 텍스트 단위 = AI 시대의 비용·과금 단위
- **거버넌스**: 누가/무엇을/얼마나 쓰는지 통제·관측·감사하는 것
